# Agent 8: Explanation Agent

**Role:** Output Formatting — Clinician Summary + Patient Communication  
**Type:** A2A Agent  
**Framework:** LangChain + FastAPI  
**Output formats:** FHIR Bundle, SOAP Note, Plain-language patient summary

---

## What This Agent Does

The Explanation Agent is the **last mile of clinical communication**. Every other agent produces technical JSON. This agent transforms all of that into two things doctors and patients actually use:

1. **For the clinician:** A structured SOAP note + FHIR Bundle containing all resources generated by the system
2. **For the patient:** A plain-language explanation of their condition, what will happen next, and why — at a 6th-grade reading level

It also applies SHAP-style feature attribution to explain *why* the Digital Twin risk model scored the patient as high-risk, translating XGBoost feature importance into human-readable sentences.

This agent is what makes MediTwin feel like a real product, not a research prototype.

---

## Responsibilities

1. Receive the complete, consensus-approved multi-agent output from the Orchestrator
2. Generate a SOAP note (Subjective, Objective, Assessment, Plan) for the clinician
3. Build a FHIR Bundle containing all generated resources (Condition, DiagnosticReport, MedicationRequest, CarePlan)
4. Generate a patient-friendly explanation (plain language, no jargon)
5. Run feature attribution on the Digital Twin's risk scores
6. Return all outputs in a structured final response

### Step 3: SHAP-Style Risk Attribution
```python
def generate_risk_attribution(digital_twin_output: dict, features: dict) -> dict:
    # Get XGBoost feature importances
    model = load_model("models/readmission_30d.json")
    feature_importances = model.get_score(importance_type='gain')
    
    # Map features to human-readable names
    feature_names = {
        "age": "Age",
        "wbc": f"WBC {features['wbc']} (critical)" if features['wbc'] > 11 else f"WBC {features['wbc']}",
        "comorbidity_count": f"Comorbidity count: {features['comorbidity_count']}",
        "has_ckd": "Chronic kidney disease",
        # ...
    }
    
    # Generate directional attribution
    attributions = []
    for feature, importance in sorted(feature_importances.items(), key=lambda x: -abs(x[1]))[:5]:
        value = features.get(feature, "N/A")
        direction = "increases_risk" if importance > 0 else "reduces_risk"
        contribution = f"+{importance:.2f}" if importance > 0 else f"{importance:.2f}"
        
        attributions.append({
            "feature": feature_names.get(feature, feature),
            "contribution": contribution,
            "direction": direction
        })
    
    return attributions
```

### Step 4: FHIR Bundle Assembly
```python
def build_fhir_bundle(all_resources: dict) -> dict:
    entries = []
    
    for resource in [
        all_resources.get("fhir_condition"),
        all_resources.get("fhir_diagnostic_report"),
        all_resources.get("fhir_medication_requests", []),
        all_resources.get("fhir_care_plan")
    ]:
        if isinstance(resource, list):
            entries.extend([{"resource": r} for r in resource if r])
        elif resource:
            entries.append({"resource": resource})
    
    return {
        "resourceType": "Bundle",
        "id": str(uuid.uuid4()),
        "type": "collection",
        "timestamp": datetime.utcnow().isoformat() + "Z",
        "entry": entries
    }
```

## Reading Level Verification

After generating patient text, verify reading level:
```python
import textstat

def verify_reading_level(text: str) -> dict:
    grade = textstat.flesch_kincaid_grade(text)
    return {
        "reading_level_grade": grade,
        "target": 6,
        "acceptable": grade <= 8,
        "score": textstat.flesch_reading_ease(text)
    }
```

If grade > 8, regenerate with stricter prompt instructions.
